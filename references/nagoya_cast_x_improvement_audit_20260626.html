キャストX改善点・悪い点指摘

総評

一番悪い点は、キャストXが「来店の空気」は作れているのに、「来店アクション」まで接続されていないこと。
今日いる、イベントがある、楽しそう、常連が盛り上がっている、という空気は出ている。 しかし、初めての人が次に何をすればよいか、どの投稿を受付で見せればよいか、LINE追加すべきか、予約が必要かが見えにくい。 そのため、投稿が認知や好感で止まりやすく、来店数として回収しづらい。

悪い点

キャスト投稿が個人の熱量に依存していて、店舗の集客導線として設計されていない。

損失

Xを見た人が「行きたいかも」で止まり、LINE追加・受付申告・初回来店に変わりにくい。

改善の核

個人の言葉は残し、末尾のCTAと計測だけ店舗側で統一する。

悪い点と改善点

優先度 悪い点 なぜ問題か 改善点
P1 キャスト投稿の末尾に明確なCTAがない 「楽しそう」「今日いる」は伝わるが、初めての人が次に取る行動が分からない。結果として、投稿が来店直前で止まる。 全員の投稿末尾に「初めての方はこの投稿を受付で見せてください」「LINEはこちら」「初心者講習できます」のどれかを入れる。
P1 投稿が計測できない どのキャスト、どの投稿型、どのイベントが来店につながったか分からない。成果が見えないと、良い投稿も再現できない。 個人名ではなく投稿型でコード化する。例: NG-FIRST、NG-EVENT、GG-WATCH、GG-DEEP。
P1 公式アカウントがキャスト投稿を受け止める設計が弱い 個人アカウントは強いが、退店・休み・投稿停止で導線が切れる。公式が受け皿にならないと資産化しない。 公式が週1で「今週の案内キャスト」「初心者講習できる日」「観戦だけOKの日」をまとめる。
P2 初心者向け情報が投稿ごとに散らばっている 初心者は「いくら」「予約」「所要時間」「一人OK」「服装」「ルール説明」を知りたい。これが毎回見えないと不安が残る。 2店舗共通の「初めての流れ」画像を作り、キャストが投稿に添付できるようにする。
P2 ナゴヤギルドとGGPL名古屋の役割分担が個人投稿上で明文化されていない 2店舗が近い関係に見える一方、初めての人はどちらに行けばよいか分からない。 「初回は名駅ナゴヤギルド」「大会挑戦は栄GGPL」のように、投稿内で階段を作る。
P2 GGPL名古屋側は専門語が強くなりやすい トーナメント、レイトレジスト、Deep、JOPT、チップ量は経験者には刺さるが、未経験者には読めない。 投稿冒頭に「初心者向け」「経験者向け」「観戦だけOK」の分類を入れる。
P2 ナゴヤギルド側は日常・出勤投稿が来店説明に接続しきっていない 人の顔は見えるが、初回来店の流れが見えないと、外部新規は動きづらい。 「今日います」に加えて「初めての方は何時から説明できます」を入れる。
P3 DM依存になりやすい DMは個人には便利だが、店舗側で計測しにくく、返信負荷も偏る。 DM相談は残してよいが、最終導線はLINE・受付・公式フォームへ戻す。
P3 プロフィール表記が統一されていない 店舗名、公式ハンドル、初心者講習可否、DM可否がばらつくと、検索で見つけても公式性が弱い。 最低限「店舗公式ハンドル」「DM可否」「初めての人向け案内の有無」を統一する。

店舗別の指摘

ナゴヤギルドの悪い点

  • 「今日いる」「イベントいる」は強いが、初めての人向けの手順説明が弱い。
  • ポーカー以外も遊べる強みがあるのに、キャスト投稿で「ポーカーを知らなくても入れる」まで毎回言えていない。
  • 旧X導線問題が残ると、キャスト投稿で興味を持った人が公式確認時に不安になる。

改善点

  • キャスト出勤投稿に「初めての人はこの投稿を見せてください」を追加。
  • 「ポーカー以外でも遊べます」をキャスト投稿に混ぜる。
  • ナゴヤギルド新公式Xへの導線を全キャストプロフィールで統一。

GGPL名古屋の悪い点

  • 大会熱は強いが、初心者には専門語が多く、参加難易度が読みにくい。
  • 観戦・飲み・スポーツ放映の入口が良いのに、毎回の型として固定されていない。
  • キャスト投稿が公式のテーブル状況やLINE追加へ接続しきっていない。

改善点

  • トーナメント投稿に「初心者向け / 経験者向け / 観戦だけOK」のラベルを付ける。
  • 観戦入口の投稿を週1で固定運用する。
  • キャスト投稿の最後に「今日のテーブル状況は公式へ」を入れる。

すぐ直すべき投稿フォーマット

用途 悪い例 改善例 なぜ改善になるか
出勤告知 「今日います」だけ 「今日16-23でいます。初めての方もルール説明できます。この投稿を受付で見せてください」 来店理由と受付申告が同時に作れる。
トーナメント告知 大会名とリンクだけ 「今日は経験者向けDeepです / 初級者も入りやすいです」を冒頭に入れる 初心者が参加可否を判断できる。
イベント告知 イベント名だけ 「ポーカーしなくてもOK。大富豪/BJ/観戦だけでも寄れます」を追加 非ポーカー層の入口になる。
観戦導線 「放映します」だけ 「観戦だけOK / 一人参加OK / 観戦後に無料初心者講習できます」 GGPLの硬さを崩し、未経験者を取り込める。
2店舗横断 各店舗が別々に投稿 「初めてはナゴヤギルド、次の大会挑戦はGGPL名古屋」 2店舗を競合ではなく成長導線にできる。

優先順位

  1. キャスト投稿の共通CTAを決める。
    まずは「初めての方はこの投稿を受付で見せてください」。これが一番早く、店頭計測にもつながる。
  2. 初めての流れ画像を2店舗分作る。
    料金、所要時間、一人OK、予約要否、LINE、声をかける相手を1枚にまとめる。
  3. 投稿型コードを作る。
    NG-FIRST、NG-EVENT、GG-WATCH、GG-DEEP、GG-JOPT。個人別ではなく用途別にする。
  4. 公式が週1でキャスト投稿を回収する。
    個人の投稿を公式が引用し、公式側の固定導線へ戻す。
  5. ナゴヤギルドとGGPL名古屋の役割を毎週見せる。
    「入口の名駅」「挑戦の栄」という分担を、キャスト投稿でも見えるようにする。
最短でやるなら。
全キャストに長いルールを配る必要はない。まずは投稿末尾の1行だけ統一する。 「初めての方はこの投稿を受付で見せてください」。この一文だけで、Xが来店計測に変わる。

やってはいけないこと

  • キャスト全員に同じ硬い宣伝文を投稿させること。個人Xの強みが消える。
  • DMだけで予約や案内を完結させること。店舗側で数字が拾えなくなる。
  • 大会名・専門語だけを増やすこと。経験者には便利だが、新規は離れる。
  • キャスト個人に集客責任を丸投げすること。公式アカウントと店頭オペレーションが受け皿になるべき。
